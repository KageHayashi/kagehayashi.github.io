Port Analysis

We run a quick nmap scan and take note of the two ports that are open, 22 - SSH and 80 - HTTP.

80 - HTTP

If we navigate to https://tenet.htb, we’ll be greeted with a Wordpress site.

Browsing through a few posts, we see mentions of Rotas and user neil comments reveals a potential sator php file and backup.

We can try navigating to http://tenet.htb/sator.php to see if there’s a file. No luck.

But if we try http://10.10.10.223/sator.php, we get some interesting results.

There was also a mention of a backup file, which we can find at http://10.10.10.223/sator.php.bak. This is a common naming scheme for backups and if we can obtain this file, we might get the source code.

Foothold

Insecure Deserialization

If we open the file, we can see the source code for sator.php.

sator.php.bak:

Taking a look at lines 23 and 24, we see that the application is taking in a parameter named arepo and deserializing it without any sanitization. This could lead to an insecure deserialization vulnerability. We can also see that the DatabaseExport class has a magic function __destruct() which creates a file with the name $this->user_file and contents $this->data. If we can control these two variables, we might be able to get arbitrary file write on the system.
